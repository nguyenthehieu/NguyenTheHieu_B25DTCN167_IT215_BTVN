Phần 1. Khái niệm cơ bản
1. Backend là gì?
Backend chính là một phần của trang web bao gồm một máy chủ, một ứng dụng và một cơ sở dữ liệu,  là người nắm giữ vị trí quan trọng trong việc đưa ra quyết định cho cách thức vận hành của một website.  
2. Client – Server là gì?
Client-server là mô hình mạng máy tính gồm có 2 thành phần chính, đó là máy khách (client) và máy chủ (server). 
Server là nơi giúp lưu trữ tài nguyên cũng như cài đặt các chương trình dịch vụ theo đúng yêu cầu của client.
Client bao gồm máy tính cũng như các loại thiết bị điện tử nói chung sẽ tiến hành gửi yêu cầu đến server.
3. API là gì?
API (Application Programming Interface) là API là cơ chế cho phép 2 thành phần phần mềm giao tiếp với nhau bằng một tập hợp các định nghĩa và giao thức. 
4. JSON là gì?
JSON (JavaScript Object Notation) là một định dạng dùng để lưu trữ và trao đổi dữ liệu. JSON có cấu trúc đơn giản, dễ đọc và được sử dụng rất phổ biến trong các ứng dụng web.

Phần 2. Kiến thức Web cơ bản
1. HTTP là gì?
HTTP (HyperText Transfer Protocol) là giao thức dùng để trao đổi dữ liệu giữa Client và Server trên Internet.
Khi người dùng truy cập một trang web, trình duyệt sẽ gửi yêu cầu HTTP đến server. Sau khi xử lý, server sẽ gửi lại dữ liệu để trình duyệt hiển thị cho người dùng.
HTTP đóng vai trò như một bộ quy tắc giúp các thiết bị trên mạng hiểu và giao tiếp với nhau.
2. Các phương thức GET, POST, PUT và DELETE
•	GET được sử dụng để lấy dữ liệu từ server.
•	POST được sử dụng để gửi dữ liệu mới lên server.
•	PUT được sử dụng để cập nhật dữ liệu đã tồn tại.
•	DELETE được sử dụng để xóa dữ liệu khỏi hệ thống.

Phần 3. Tìm hiểu FastAPI
1. FastAPI là gì?
FastAPI là một Web Framework được xây dựng bằng ngôn ngữ Python để phát triển các Web API hiện đại.
Framework này được thiết kế với mục tiêu đơn giản, dễ sử dụng và có hiệu năng cao. FastAPI hỗ trợ lập trình bất đồng bộ (Asynchronous Programming) nên có khả năng xử lý nhiều yêu cầu cùng lúc.
2. FastAPI dùng để làm gì?
FastAPI thường được sử dụng để xây dựng các API cho:
•	Website
•	Ứng dụng di động
•	Hệ thống quản lý dữ liệu
•	Dịch vụ Microservices
•	Các ứng dụng tích hợp trí tuệ nhân tạo
FastAPI giúp lập trình viên xây dựng API nhanh chóng và dễ bảo trì.
3. FastAPI khác gì so với các Web Framework truyền thống?
Các Framework truyền thống thường hoạt động theo mô hình đồng bộ nên khi số lượng yêu cầu tăng cao có thể tiêu tốn nhiều tài nguyên hơn. Trong khi đó, FastAPI được xây dựng dựa trên chuẩn ASGI nên có thể xử lý nhiều kết nối đồng thời hiệu quả hơn. Ngoài ra, FastAPI còn có khả năng tự động tạo tài liệu API, giúp việc kiểm thử và phát triển thuận tiện hơn. Cú pháp của FastAPI cũng khá đơn giản, dễ tiếp cận đối với người mới học Python.
